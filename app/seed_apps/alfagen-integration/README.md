# AlfaGen Sandbox

Интеграция с корпоративным LLM-сервисом AlfaGen Sandbox (Альфа-Банк). Позволяет:

- Запускать чаты с GenAI-моделями (Qwen, Llama, gpt-oss)
- Загружать документы для RAG (PDF, DOCX, TXT, …)
- Автоматически улучшать описания найденных дефектов
- Проводить анализ результатов запусков

## Настройка

1. Отправь письмо на `EYanovskaya@alfabank.ru` с темой «Получение доступа к Sandbox (ФИО)», приложи заполненную анкету. Срок выдачи — ~2 рабочих дня.
2. В ответе придёт авторизационный ключ в формате UUID.
3. В настройках установленного приложения укажи:
   - **URL Sandbox API** — по умолчанию `https://agenapisandbox.moscow.alfaintra.net`
   - **Авторизационный ключ** — UUID из письма
   - **systemId** — всегда `sanduser` для Sandbox
   - **Модель по умолчанию** — одна из 6 доступных
   - **Улучшать описания дефектов** — если нужно авто-обогащение
   - **Промпт для обогащения** — как должна переформулировать модель
   - **Обогащать приоритеты** — какие дефекты обогащать автоматически

## Возможности

### 💬 Чат

Открой **AlfaGen** в боковом меню → вкладка «Чат». Выбери модель, общайся. Поддерживается прикрепление файлов, загруженных во вкладку «Файлы / RAG».

### 📄 Файлы / RAG

Вкладка «Файлы / RAG» позволяет загружать документы: PDF, DOCX, TXT и т.д. После обработки файл получает `file_id` и может быть прикреплён к любому сообщению в чате.

### 🤖 Автообогащение дефектов

Когда агент сохраняет дефект с приоритетом из `auto_enrich_priorities`, хэндлер `builtin:alfagen.enrich_defect` отправляет текст дефекта в AlfaGen с заданным промптом и получает улучшенное описание — структурированное по ГОСТ. Результат сохраняется в `defect.llm_analysis_json.alfagen_enriched`.

### 📊 Анализ запуска

На странице результатов любого запуска — кнопка «Анализ через AlfaGen». Она собирает данные (экраны, переходы, статус), передаёт в выбранную модель и отдаёт QA-разбор: резюме, разбор дефектов, оценка покрытия.

## Ограничения Sandbox

- Доступ только из сети **VPN / VDI Альфа-Банка**. Локальная разработка — через VPN.
- Лимит: **0.2 RPS** (1 запрос в 5 секунд)
- Срок доступа: **3 месяца** с момента выдачи ключа
- Нельзя подключаться из production системы (для этого — AlfaGen API)

## Как устроено

Приложение использует встроенный (`builtin:`) механизм исполнения:

- Markov-бэкенд проксирует все запросы к AlfaGen через `/api/apps/builtins/alfagen/*`. Токен и systemId хранятся в `installation.settings` и не покидают сервер.
- iframe приложения получает короткоживущий installation-токен на 1 час и обращается к этим proxy endpoints.
- Hook `defect.created` диспатчится в `app.services.app_builtins.alfagen_enrich_defect`.

## Логи доставки

Все события, доставленные в этот handler, пишутся в таблицу `app_event_deliveries`. Ошибки видны в логе backend-контейнера.
