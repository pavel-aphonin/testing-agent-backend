# Jira интеграция

Автоматически заводит баги в Jira из дефектов, найденных агентом. Позволяет вручную создавать issue из страницы результатов запуска.

## Настройка

1. В Jira Cloud создай API-токен: **Profile → Security → Create API token**.
2. В настройках установленного приложения укажи:
   - **URL Jira**: `https://<твой-домен>.atlassian.net`
   - **Email для API**: твой email в Jira
   - **API-токен**: созданный токен
   - **Ключ проекта**: например `MARKOV` или `QA`
   - **Соответствие приоритетов**: JSON-маппинг наших приоритетов в Jira. По умолчанию:
     ```json
     {"P0":"Highest","P1":"High","P2":"Medium","P3":"Low"}
     ```
   - **Автосоздавать для**: `P0,P1` — только критичные дефекты автоматически попадают в Jira.
3. В боковом меню появится пункт «Jira». Открой его и нажми **Проверить** чтобы убедиться что настройки верны.

## Как работает автосоздание

Когда агент сохраняет дефект, Марков эмитит событие `defect.created`. Наш хэндлер читает payload, конвертирует приоритет по маппингу и POST-ит в `<jira_url>/rest/api/2/issue`.

Если приоритет дефекта не в списке `auto_create_priorities` — issue не создаётся автоматически. Его всегда можно создать вручную с кнопки «Создать в Jira» на странице результатов запуска.

## Ручное создание

Откой любой завершённый запуск → кнопка **Создать в Jira** в заголовке → форма, в которой уже заполнен `run_id` в контексте.
